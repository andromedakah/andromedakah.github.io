Who Goes There?

Saturday, 4 July 2026 · Cross-sector edition · C-level + Engineering

The fastest-growing workforce in your company has no badge. 2026's defining security story isn't a new exploit — it's that your AI agents act with human-level access and no human identity.

1 · Executive Summary

For two years the question about agents was can they do the work? This week the board faces a sharper one: who are they, and what are they allowed to touch? Adoption has raced ahead of control — the gap now has a name (the enforcement gap) and a number.

1. Adoption has outrun control. 88% of orgs reported a confirmed/suspected AI-agent incident last year — yet 82% of execs believe their agents are protected. That spread is the enforcement gap. Healthcare: 92.7%.

2. Privileges, but no identity. Only 21.9% treat agents as independent identities; only 7.2% have a named human accountable; only 14.4% got full security sign-off. Machines outnumber humans 82:1; 68% lack identity controls for AI (CyberArk).

3. Governance is the new gate. Microsoft began rolling out Entra Conditional Access + ID Protection for Agents in early July 2026 — Zero Trust extended to agents. The platform message: an agent with no identity is a liability, not a feature.

Bottom line: the question moves from how many agents can we deploy to can we prove who each one is and bound what it can do.

2 · Allegory — The Master Key

A firm badged every employee — the lobby impressed auditors. Then it hired a thousand temps overnight and, with no time to badge them, left a single master key under the mat. It opened every door. One night a temp opened one it shouldn't — and no one could say which temp, because they'd all shared one untraceable key.

The fix wasn't firing the temps. It was badging them too — a distinct, scoped, expiring credential per worker. That badge is agent identity.

The question it forces: If an agent did something tomorrow it shouldn't, could we name which one, on whose authority, and shut only it off — in minutes, not weeks?

3 · Technical — Zero Trust for Non-Human Identities

The fastest-growing "user" population isn't human (machines outnumber people ~82:1). Yet most agents run on a service account, a shared API key, or an inherited human's OAuth scopes — dangerous once a machine reasons, chains tools, and can be steered by untrusted input. Five controls fix it:

Distinct identityOne nameable principal per agent — every action attributes to exactly one.
Least privilegeOnly what the task needs, not everything its human sponsor can do.
Short-lived credsFast-expiring, instantly revocable — no standing master key.
Policy gateEvaluate this agent, action, and context on every call — MFA for agents.
Runtime auditEvery tool call logged — an agent can be turned mid-task.

On legacy: identity infra was built for humans (days to provision, quarterly reviews). Agents are created in seconds, exist in thousands, act at machine speed. The retrofit is an identity fabric / agent gateway issuing a distinct short-lived credential per agent, enforcing least privilege per call, and capturing the audit trail legacy apps never produced.

5 · So What — 3 Moves for the Next 30 Days

1. Inventory agents as identities, not integrations. One list: every production agent, its credential, entitlements, and named human sponsor.

2. Kill the shared keys. Move agents onto distinct, least-privilege, expiring credentials via an identity fabric / agent gateway; pilot vendor Zero Trust on your highest-privilege agents first.

3. Close the enforcement gap. Report a monitoring-coverage number to the board next to adoption; log every tool call; rehearse revoking a single agent in minutes.
